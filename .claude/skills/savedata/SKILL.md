# savedata - セッション内容の Acta デイリーノート保存

現在の会話（セッション）の内容を要約し、Acta のデイリーノートに acta:comment ブロックとして追記する。

## 保存先

```
${HOME}/ghq/github.com/inamuu/data/Acta/posts/yyyy/mm/dd/yyyy-mm-dd.md
```

- `yyyy` / `mm` / `dd` は保存日の年・月・日（例: `2026/07/16/2026-07-16.md`）
- 日ディレクトリが無ければ `mkdir -p` で作成する
- デイリーファイルが無ければ先頭行を `# yyyy-mm-dd` として新規作成する
- デイリーファイルが既にあれば末尾に追記する（既存内容は変更しない）
- パスは必ず `$HOME` から解決すること（他PCでも共用するためハードコード禁止）
- 注意: `posts/yyyy/mm/` 直下に置いた素の md ファイルは Acta に読み込まれない。必ず日ディレクトリ内のデイリーファイルに acta:comment ブロックで追記すること

## 手順

1. 日付・uuid・エポックミリ秒を取得する

   ```bash
   date +%Y/%m/%d && date "+%Y-%m-%d %H:%M"
   python3 -c "import uuid,time; print(uuid.uuid4()); print(int(time.time()*1000))"
   ```

2. 保存先の日ディレクトリを `mkdir -p` で作成し、デイリーファイルの有無を確認する
3. 会話全体を振り返り、下記フォーマットの acta:comment ブロックをデイリーファイル末尾に追記する（新規作成時は `# yyyy-mm-dd` 見出しの後に置く）
4. 保存後、保存先のフルパスをユーザーに報告する

## acta:comment ブロックのフォーマット

```markdown

<!-- acta:comment
id: <uuidgen 相当の UUID>
created: yyyy-mm-dd HH:MM
created_ms: <エポックミリ秒>
tags: 
-->
## <セッションのタイトル（日本語）>

<セッションで何をしたか、2〜5行の概要。作業ディレクトリ・関連リポジトリもここに書く>

### 実施内容

<実際に行った作業を箇条書きで。変更したファイル、実行したコマンド、調査結果など>

### 成果物

<作成・変更したファイルのパス、PR、生成物など。無ければ省略>

### 決定事項・学び

<決めたこと、判明したこと、今後に活きる知見。無ければ省略>

### 未対応・TODO

<やり残しや次のアクション。無ければ省略>
<!-- /acta:comment -->
```

- `id` / `created` / `created_ms` は追記のたびに新しく生成する
- `tags:` は空のままでよい（ユーザーから指定があれば設定する）
- ブロック内の見出しは `##` から始める（デイリーファイルのタイトルが `#` のため）

## 注意

- 出力は日本語、絵文字なし
- シークレット（トークン、パスワード、APIキー等）は記録しない
- 会話ログの全文コピーではなく、後から読んで役に立つ要約にする
