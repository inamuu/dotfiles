# savedata - セッション内容のMarkdown保存

現在の会話（セッション）の内容をMarkdownにまとめ、月単位のディレクトリに保存する。

## 保存先

```
${HOME}/Documents/Claude/Summarize/yyyy/mm/
```

- `yyyy` / `mm` は保存日の年・月（例: `2026/07/`）
- ディレクトリが無ければ `mkdir -p` で作成する
- パスは必ず `$HOME` から解決すること（他PCでも共用するためハードコード禁止）

## ファイル名

```
yyyy-mm-dd_<トピックのスラッグ>.md
```

- 例: `2026-07-08_terraform-s3-refactor.md`
- スラッグはセッションの主題を英小文字ケバブケースで短く（3〜5語程度）
- 同名ファイルが既に存在する場合は `_2`, `_3` を付けて衝突を回避する

## 手順

1. `date +%Y/%m` と `date +%Y-%m-%d` で保存日を取得する
2. 保存先ディレクトリを `mkdir -p` で作成する
3. 会話全体を振り返り、下記フォーマットでMarkdownを作成して保存する
4. 保存後、保存先のフルパスをユーザーに報告する

## Markdownフォーマット

```markdown
# <セッションのタイトル（日本語）>

- 日時: yyyy-mm-dd
- 作業ディレクトリ: <cwd>
- 関連リポジトリ: <あれば>

## 概要

<セッションで何をしたか、2〜5行で>

## 実施内容

<実際に行った作業を箇条書きで。変更したファイル、実行したコマンド、調査結果など>

## 成果物

<作成・変更したファイルのパス、PR、生成物など。無ければ省略>

## 決定事項・学び

<決めたこと、判明したこと、今後に活きる知見。無ければ省略>

## 未対応・TODO

<やり残しや次のアクション。無ければ省略>
```

## 注意

- 出力は日本語、絵文字なし
- シークレット（トークン、パスワード、APIキー等）は記録しない
- 会話ログの全文コピーではなく、後から読んで役に立つ要約にする
