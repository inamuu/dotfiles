# Terraform Refactor Advisor

Terraform コードの品質を改善するためのリファクタリング提案を行うスキルです。

## 入力

- Terraform コード (.tf ファイルの内容)
- またはモジュールのディレクトリ構造

引数がない場合は「リファクタリング対象の Terraform コードを貼り付けてください」と聞く。

## 検出・提案項目

### count → for_each 変換
- `count` でリソースを複数作成している場合、`for_each` への変換を提案
- `count.index` で名前に番号を付けているパターンを検出

### モジュール化
- 同一構成のリソース定義が3回以上繰り返されている場合にモジュール化を提案
- 典型例: ECS タスク定義、セキュリティグループ、IAM ロール

### locals の整理
- 同じ値が複数箇所でハードコードされている場合に `locals` ブロックへの抽出を提案
- `local.` 参照で重複定義されている locals を統合提案

### 変数化
- ハードコードされた以下の値を検出して変数化を提案:
  - インスタンスタイプ
  - リージョン
  - 環境名 (prod/staging 等)
  - CIDR ブロック

### 命名・タグ
- タグの `Name` / `Env` が未設定のリソースを検出
- 命名規則が統一されていない場合に指摘

### lifecycle
- `prevent_destroy` が設定されていないステートフルリソース (RDS, ElastiCache) を検出
- `create_before_destroy` が有効な場合の注意点

### 依存関係
- 暗黙的依存ではなく `depends_on` が必要な場合を検出

## 出力フォーマット

```
## Terraform Refactor Suggestions

### 優先度: 高
1. (問題) → (提案)
   \`\`\`hcl
   # Before
   ...
   # After
   ...
   \`\`\`

### 優先度: 中
2. (問題) → (提案)

### 優先度: 低
3. (問題) → (提案)
```

## 注意

- コードの意図を変えないリファクタリングのみ提案する
- Before/After のコード例を必ず示す
- 出力は日本語
