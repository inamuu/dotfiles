# PR Summary Generator

大規模な Pull Request を要約するスキルです。

## 入力

以下のいずれか:
- GitHub PR の URL
- diff テキスト
- 変更ファイル一覧とコードの抜粋

引数がない場合は「要約したい PR の URL または diff を貼り付けてください」と聞く。

## 解析内容

以下を分析する:

- 変更されたファイル数・行数
- インフラ変更 (Terraform, CloudFormation, k8s manifest)
- アプリケーション変更 (ロジック変更・API 変更・設定変更)
- 依存関係の変更 (package.json, go.mod, requirements.txt など)
- テスト変更
- CI/CD パイプライン変更
- セキュリティ関連の変更

## リスク評価基準

| リスク | 条件 |
|--------|------|
| High | インフラ削除・DB スキーマ変更・認証ロジック変更・本番設定変更 |
| Medium | インフラ追加・API インターフェース変更・依存関係メジャーアップ |
| Low | バグ修正・ドキュメント・リファクタリング・テスト追加 |

## 出力フォーマット

```
## PR Summary

### 変更概要
(1〜3行で変更の目的と内容)

### インフラ変更
- (変更点を箇条書き。なければ「なし」)

### アプリケーション変更
- (変更点を箇条書き。なければ「なし」)

### 依存関係変更
- (変更点を箇条書き。なければ「なし」)

### リスク評価
**レベル: Low / Medium / High**
- (リスクの理由)

### レビュー重点箇所
1. (特に注意してレビューすべき箇所)
2. ...
```

## 注意

- GitHub PR URL が渡された場合は `gh pr diff` または WebFetch で内容を取得してから解析する
- 出力は日本語
