# Summary Weeks (Acta)

このコマンドは、Acta の指定期間のアクティビティをサマリーにして、今日のファイルに追記するための専用コマンドです。

## 対象ディレクトリ
`$HOME/ghq/github.com/inamuu/data/Acta/`

**注意**: パスは `$HOME` 環境変数を使用するため、どのMacでも動作します。

## 目的
1. 指定期間のActaファイル（デフォルト5日分）を読み込む
2. 各日のアクティビティを集約してサマリーを作成する
3. 今日の日付のファイルに `acta:comment` ブロックとして追記する
4. 適切なタグを自動で付与する

## 実行手順

### 1. 期間の決定
- デフォルト: 今日を含めた過去5日分
- ユーザーが引数で日数を指定した場合はその日数分（例: `/summary-weeks 7` で7日分）
- 日本時間（`Asia/Tokyo`）で日付を計算する
- **実装ヒント**:
  - 今日: `TZ=Asia/Tokyo date '+%Y-%m-%d'`
  - N日前: `TZ=Asia/Tokyo date -v-Nd '+%Y-%m-%d'` (macOS)

### 2. ファイルの読み込み
- 対象期間の各日付に対応するファイル（例: `2026-02-22.md`, `2026-02-23.md`, ...）を読み込む
- ファイルが存在しない日はスキップする
- 全ての `acta:comment` ブロックを抽出する

### 3. サマリーの生成
以下の構造でサマリーを作成する:

```markdown
## 📊 YYYY年MM月DD日〜DD日 活動サマリー

### 主要プロジェクト

#### [プロジェクト名]

**[具体的なタスク名]** (実施日)
- 要点1
- 要点2

### 開発環境・ツール改善

**[改善内容]** (実施日)
- 要点

### その他の活動

- 活動内容

### 進捗ステータス

**✅ 完了**: ...

**🔄 進行中**: ...

**⏳ 未着手**: ...
```

**サマリー作成のガイドライン**:
- **サマリー全体は20行以内に収める**
- 複数日にわたる同じプロジェクト/タスクは統合する
- 各項目は箇条書きで簡潔にまとめ、詳細な説明は省く
- 技術的な詳細（PR番号、コマンド、エラー名など）は残す
- 進捗ステータスは全体を俯瞰して整理する
- 絵文字は控えめに使用（セクション見出しのみ）

**タグの自動抽出**:
- 全ファイルから `tags:` フィールドを収集する
- 出現頻度の高いタグを特定する
- 主要なタグを自動選択してサマリーに付与する

### 4. 今日のファイルへの追記
- ファイル: `$HOME/ghq/github.com/inamuu/data/Acta/{today_ymd}.md`
- ファイルがない場合は、先頭に `# {today_ymd}` を作成してから追記する
- `acta:comment` ブロックとして保存する
- メタデータ:
  - `id`: UUID v4（**実装ヒント**: `uuidgen | tr '[:upper:]' '[:lower:]'`）
  - `created`: `YYYY-MM-DD HH:MM`（**実装ヒント**: `TZ=Asia/Tokyo date '+%Y-%m-%d %H:%M'`）
  - `created_ms`: Unix timestamp (milliseconds)（**実装ヒント**: `TZ=Asia/Tokyo date '+%s000'`）
  - `tags`: 自動抽出したタグ + 固定で `Summary, WeeklySummary` を含める

**タグ付与ルール**:
- 必須タグ: `Summary`, `WeeklySummary`
- 頻出タグ: 期間中に3回以上出現したタグを追加（最大5個まで）
- タグ優先順位: `ToDo_Medley` > `Subworks_*` > `JM_*` > その他
- 形式: `tags: Summary, WeeklySummary, ToDo_Medley, Subworks_ohtani_design`

### 5. 出力
- 追記したファイルパスを表示
- サマリーの対象期間を表示
- 抽出したタグを表示

## 出力ルール
- 返答は日本語
- 変更したファイルパスと実施内容を短く示す
- サマリーは見やすく構造化する
- 専門用語や技術用語はそのまま記載する

## 実装の効率化ポイント
1. **並列読み込み**: 対象期間のファイルを並列で読み込む（複数の `Read` ツールを同時実行）
2. **タグ集計**: ファイル読み込み時に同時にタグを収集・カウントする
3. **メタデータ生成**: UUID、日時、タイムスタンプを並列で取得する

## 使用例
```
/summary-weeks          # デフォルト5日分のサマリーを作成
/summary-weeks 7        # 7日分のサマリーを作成
/summary-weeks 14       # 14日分（2週間）のサマリーを作成
```
