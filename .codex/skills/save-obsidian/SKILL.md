# Obsidian Vault Save Skill

会話中の調査結果やレポートをObsidian Vaultに保存する。

## 保存先ベースパス

`/Users/kazuma.inamura/ghq/github.com/inamuu/obsidian/Vaults/Agent/`

## ディレクトリ構成と用途

| ディレクトリ | 用途 | 保存する内容 |
|---|---|---|
| `outputs/` | AIの成果物（レポート・スライド等） | 調査レポート、まとめ、Marpスライド、可視化画像 |
| `logs/` | 調査ログ・実行メモ | AIとのやり取りの記録、調査経過メモ、要確認事項リスト |
| `wiki/` | 概念ノート・要約 | 記事の要約、概念整理ノート、比較表、分類 |
| `raw/` | 生データ | 加工前の元データ |

## 実行手順

1. **保存する内容の種別を判断**
   - 調査結果・レポート → `outputs/`
   - 作業ログ・メモ → `logs/`
   - 概念整理・要約 → `wiki/`
   - 生データ → `raw/`

2. **ファイル名を決定**
   - 内容を端的に表す英語のケバブケース（例: `jm-qa-s3-acl-issue.md`）
   - 日付が重要な場合はプレフィックスに付与（例: `2026-04-06-deploy-failure.md`）

3. **Markdown形式で保存**
   - 見出し、リスト、テーブル、コードブロックを適切に使用
   - 最上位見出し（`#`）にタイトルを記載

4. **保存完了を報告**
   - 保存先のフルパスを表示

## 重要な注意点

- 既存ファイルがある場合は上書き前にユーザーに確認する
- 機密情報（クレデンシャル、APIキーなど）は保存しない
- Obsidianで読みやすいMarkdown形式を心がける
