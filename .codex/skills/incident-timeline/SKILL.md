# Incident Timeline Generator

ポストモーテム用のインシデントタイムラインを生成するスキルです。

## 入力

以下の1つ以上:
- ログ (アプリ・インフラ・アクセスログ)
- アラート通知
- モニタリング出力 (CloudWatch、Datadog、Grafana など)
- Slack のスレッドや通話メモ
- 操作履歴

引数がない場合は「タイムライン作成に使いたいログや情報を貼り付けてください」と聞く。

## 解析内容

以下を時系列順に整理する:

1. **検出**: アラート発火・異常検知の最初の時刻
2. **影響開始**: ユーザー影響や機能停止が始まった時刻
3. **エスカレーション**: 担当者へのアサイン・会議開始
4. **原因特定**: 根本原因が判明した時刻
5. **対処開始**: 修正・回避策の実施開始
6. **回復**: サービスが正常状態に戻った時刻
7. **影響終了**: ユーザー影響が完全に解消された時刻

## 出力フォーマット

```
## Incident Timeline

**概要**: (1行でインシデントの概要)
**影響時間**: HH:MM 〜 HH:MM (XX分間)

### タイムライン

| 時刻 | イベント | 担当 |
|------|---------|------|
| HH:MM | (イベント内容) | (担当者/チーム) |
| HH:MM | ... | ... |

### フェーズ別サマリー

- **検出**: HH:MM
- **影響開始**: HH:MM
- **原因特定**: HH:MM
- **回復**: HH:MM
- **MTTR**: XX分

### ポストモーテム用メモ
- 何が起きたか:
- なぜ起きたか:
- どう解決したか:
- 次回の予防策:
```

## 注意

- タイムスタンプは入力に合わせた形式を維持する (JST/UTC を明記)
- 担当者情報がない場合は省略する
- 出力は日本語
