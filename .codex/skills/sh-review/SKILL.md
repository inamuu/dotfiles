# Shell Script Reviewer

シェルスクリプトのコードレビューを行うスキルです。

## 入力

以下のいずれか:
- シェルスクリプトのファイルパス
- シェルスクリプトのコード片

引数がない場合は「レビュー対象のシェルスクリプトのパスまたはコードを貼り付けてください」と聞く。

## 解析項目

### 1. 構文・バグ

以下を検出する:
- シェル構文の誤り、括弧やクォートの不一致
- 未定義変数の参照
- クォートされていない変数展開 (`$var` → `"$var"`)
- `cd` の戻り値未チェック (`cd dir || exit 1` 推奨)
- パイプの途中エラーの見落とし (`set -o pipefail` 未設定)
- `set -euo pipefail` の欠如
- word splitting / glob 展開の意図しない発生

### 2. セキュリティ

以下を検出する:
- コマンドインジェクションの可能性 (`eval`, バッククォート内の未検証入力)
- パストラバーサルの脆弱性
- 安全でない一時ファイル作成 (`/tmp/固定名` → `mktemp` 推奨)
- 機密情報のハードコード (パスワード、トークン、APIキー)
- 過剰なパーミッション設定 (`chmod 777`)
- `curl | bash` パターン

### 3. 移植性・互換性

以下を検出する:
- シバン行の欠如または不適切な指定 (`#!/usr/bin/env bash` 推奨)
- Bash 固有機能を `#!/bin/sh` で使用
- GNU 拡張コマンドへの依存 (macOS 非互換)
- 未宣言の外部コマンド依存

### 4. パフォーマンス

以下を検出する:
- 不要なサブシェル起動
- 非効率なループ処理 (ループ内での外部コマンド過剰呼び出し)
- パイプラインの最適化余地

### 5. 可読性・保守性

以下を検出する:
- 関数未使用の長大な処理
- マジックナンバー・ハードコードされたパス
- エラーメッセージの欠如 (サイレント失敗)
- 不適切な exit code (`exit 0` で異常終了)
- `[` と `[[` の混在
- 変数名の明確性

## 重要度の判定基準

**Critical:**
- セキュリティ脆弱性、データ損失の可能性、構文エラー

**Warning:**
- エラーハンドリング不足、ベストプラクティス違反、パフォーマンス問題、移植性の問題

**Info:**
- コードスタイル、可読性の改善、より良いアプローチの提案

## 出力フォーマット

以下のマークダウン形式で出力する:

```
## Shell Script Review

### 概要
(スクリプトの目的を一言で)

### Critical
(即座に修正が必要な問題。ファイル名:行番号、問題、修正案を記載。なければ「なし」)

### Warning
(修正を推奨する問題。ファイル名:行番号、問題、修正案を記載。なければ「なし」)

### Info
(より良いコードのための提案。なければ「なし」)

### Summary
- Total Issues: X件 (Critical: X, Warning: X, Info: X)
- Overall Assessment: Good / Needs Improvement / Critical Issues Found
```

## 注意

- ファイルパスが渡された場合は Read ツールで内容を読み取ってから解析する
- shellcheck 相当のチェックを意識する
- 必ず具体的な修正案を提示する
- 問題がない項目は「なし」と簡潔に記載する
- 出力は日本語
